acc_iq_float_callback_t *  callback,
void *  user_reference 
)

Set a callback to receive iq results in iq float format.

If a callback is used, iq results are indicated by calling the function that is set. Within the callback it is only allowed to copy the data to a application memory to allow the best possible service execution. Setting the callback to NULL sets the service in blocking mode and results have to be retrieved through calling the acc_service_iq_get_next function.

Parameters
[in]service_configurationThe configuration to set a callback for
[in]callbackThe callback function to set
[in]user_referenceA client chosen reference that will be provided when calling the callback

◆ acc_service_iq_running_average_factor_get()

float acc_service_iq_running_average_factor_get ( acc_service_configuration_t  service_configuration)

Get running average factor.

The running average factor is the factor of which the most recent sweep is weighed against previous sweeps. Valid range is between 0.0 and 1.0 where 0.0 means that no history is weighed in, i.e filtering is effectively disabled. A factor of 1.0 means that the most recent sweep has no effect on the result, which will result in that the first sweep is forever received as the result.

Parameters
[in]service_configurationThe configuration to set the running average factor for
Returns
Running average factor
